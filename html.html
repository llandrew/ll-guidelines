General

  • Always use double quotes ("") instead of single quotes ('') for attributes.
  • Don't include a trailing slash in self-closing elements.
  • Don't omit option closing tags (e.g. </li>).
  • Always indent child elements (two spaces).

Keep your code simple

Keep markup as simple as possible to fit the need. Avoid using extra elements for styling purposes. Avoid having extraneous parent elements or unneccessary complexity.

Bad...


<div class="image__container">
  <span class="image__image__container">
    <img src="http://...">
  </span>
  <span class="image__caption__container">
    <p class="image__caption__content">
      ...
    </p>
  </span>
</div>

Better


<div class="image__container">
  <img src="http://...">
  <p class="image__caption">
    ...
  </p>
</div>

Javascript generated markup

Having Javascript generate markup makes maintaining templates more difficult, so try to avoid it whenever possible.

Doctype

To make sure standards mode is enforced and that there is a more consistent rendering across browsers, a doctype must be declared at the beginning of every HTML page.


<!DOCTYPE html>
<html>
  ...
</html>

Language Attribute

Declaring a language through the lang attribute is encouraged as it promotes accessability by aiding speech synthesis and translation tools.


<html lang="en-us">
  ...
</html>

IE Compatibility Mode

Unless there is a reason to have IE render as a specific version, the <meta> tag should always be set to the latest supported mode with edge mode.

Read More 

<meta http-equiv="X-UA-Compatible" content="IE=Edge">

Character Encoding

For the sake of consistent rendering of text, it is important to always set the character encoding <meta> tag.


<meta charset="UTF-8">

Asset Includes

CSS includes should use the <link> tag and be included in the head of the file. If inline styles are needed, they should wrapped in a <style> tag and also included in the head.

When including JS files, always try to use a <script> tag with a src set and have it included just before the closing <body> tag.

With HTML5, there is no need to specify a type when including CSS or JS files as the defaults for <link> and <script> are already set.


<head>
  <link rel="stylesheet" href=".../style.css">

  <style>
    ...
  </style>

  <script src=".../script.js"></script>
</head>

Attributes

Attribute order

When writing HTML, attributes should be listed by order of specificity.

The ideal order is following:

  • class, type
  • id, name
  • data-*
  • src, for, type, href, value
  • title, alt
  • aria-*, role

<a class="..." id="..." data-modal="toggle" href="#">
  ...
</a>

<input type="text" class="form-control">

<img src="..." alt="...">

Boolean attributes

With the introduction of HTML5, boolean attributes (an attribute that doesn't need a declared value) are no longer required to have declared values to be compliant with standards.

The presence of a boolean attribute on an element represents the true value, and the absence of the attribute represents the false value.
[...]
If the attribute is present, its value must either be the empty string or [...] the attribute's canonical name, with no leading or trailing whitespace. 

<input type="text" disabled>

<input type="checkbox" value="1" checked>

<select>
  <option value="1" selected>1</option>
</select>

Structure

The new, more semenatic HTML elements allow for better readability and cleaner structures. Always try to use the most appropriate block-level element for main layout sections, reserving <div> elements for more specific sections.

Good...


<div class="header">
  <div class="nav">
    ...
  </div>
  <div class="content">
    <div class="content__section">
      ...
    </div>
  </div>
</div>

Better


<header class="header">
  <nav class="navigation">
    ...
  </nav>
</header>
<main class="content">
  <section class="content__section">
    ...
  </section>
</main>

Commenting

Use comments to mark closing tags for large, block-level elements.
